Processing a Publication Manifest



Description

This is a non-production level tool to see what transformations are performed by the Processing Algorithm of the Publication Manifest specification. The internal code is based on an test level code of the algorithm, also available on github.

The user may provide a JSON-LD file containing a Publication Manifest, which is transformed into the internal data structure following the specification. The result is displayed in Yaml for an easier read. The possible ways of submitting a manifest are:

  • The text panel on the left, labeled as “JSON-LD manifest to process” is editable: the user can type in, or paste, a full manifest. Pushing the “Process” button generates the Yaml representation of the generation result. The panel can also be filled by:
    • providing a URL for either a JSON-LD manifest file or an HTML file with embedded or linked manifest; or
    • uploading a local file.

Note that, strictly speaking, some of those combination may not be correct: indeed, the processing algorithm requires that all relative URL-s are transformed into absolute ones. This cannot be done properly if the manifest is copy pasted, edited by hand, or uploaded from a local file; to make the result formally correct the base value of https://www.example.org/pm/ is used for that case.

The page uses API-s and HTML attributes that may not be available for all browsers yet.

The tool is based on a (“browserified”) Typescript processor. The tool can also be installed via standard npm tools and ran locally through a simple CLI.
